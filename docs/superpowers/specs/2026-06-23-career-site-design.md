# Карьерный сайт ВсеИнструменты.ру — дизайн

**Дата:** 2026-06-23
**Статус:** утверждён

## Цель

Учебный/демо карьерный сайт компании ВсеИнструменты.ру («Работай у нас»),
опубликованный на GitHub, чтобы автор (новичок) учился работать с git:
делать ветки, коммиты и пуши, а сайт был доступен онлайн.

## Аудитория автора

Новичок в вёрстке и git. Все шаги сопровождаются объяснениями.

## Технический подход

Статический сайт на чистых HTML + CSS (+ немного JS), без сборщиков и
фреймворков. Бесплатный хостинг — GitHub Pages. Данные вакансий
захардкожены (это демо). Форма отклика — без бэкенда, через `mailto:`.

Отвергнутые альтернативы:
- **Tailwind через CDN** — красивее, но лишний синтаксис для новичка.
- **Фреймворк (Astro/Next.js)** — требует Node.js и сборки, перебор для задачи.

## Структура файлов

```
Test1_career/
├── index.html        вся страница с секциями-якорями
├── styles.css        все стили
├── script.js         меню, плавная прокрутка
├── images/           логотип, фоновые изображения
├── .gitignore        исключения для git
└── README.md         описание проекта
```

## Секции страницы (одностраничник)

1. **Шапка (hero):** название, слоган «Работай в ВсеИнструменты.ру»,
   кнопка-ссылка «Смотреть вакансии» (якорь на секцию вакансий).
2. **О компании / почему мы:** 3–4 карточки с преимуществами.
3. **Вакансии:** 4–5 карточек-примеров (должность, город, формат,
   кнопка «Откликнуться» → mailto).
4. **Контакты / отклик:** контакты HR + кнопка «Откликнуться» (mailto).
5. **Подвал:** ссылки, копирайт.

## Git / GitHub workflow (ключевая цель)

Настраивается с пошаговыми объяснениями:

1. `git init` — инициализация репозитория в папке.
2. Первый коммит с готовым сайтом.
3. Создание репозитория на GitHub (через сайт github.com вручную, т.к.
   `gh` CLI не установлен) и привязка через `git remote add`.
4. Пуш в ветку `main`.
5. Демонстрация работы с ветками: `git checkout -b <ветка>`, правки,
   пуш, слияние через Pull Request.

## Публикация

GitHub Pages включается в настройках репозитория. Сайт доступен по
адресу `<имя>.github.io/Test1_career`. Пуш в `main` обновляет сайт.

## Проверка

- Локально: открыть `index.html` в браузере на каждом этапе.
- После публикации: проверить живую ссылку GitHub Pages.

## Открытые вопросы (решить в плане)

- Ставить ли `gh` CLI или создавать репозиторий через веб-интерфейс
  github.com (по умолчанию — веб, как нагляднее новичку).

## Вне рамок (YAGNI)

- Настоящий бэкенд и база вакансий.
- Реальная форма с сохранением откликов (пока mailto).
- Раздел «Льготы» отдельной секцией (преимущества входят в «Почему мы»).
- Адаптация под фреймворк/Tailwind (возможный будущий шаг).
